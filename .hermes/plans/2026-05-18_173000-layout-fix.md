# Compendium Browser Layout Fix Plan

> **Goal:** Make the Compendium Browser match the dnd5e reference layout from screenshot 2.

**Current state:** Screenshot 1 â€” scattered icons, no filter sidebar, redundant title, broken grid.
**Target state:** Screenshot 2 â€” clean vertical tabs, unified filter sidebar, proper results grid.

---

## Root Cause Analysis

### R1: No `.sidebar` container exists in DOM
The PARTS definition uses `container: { id: "sidebar", classes: ["sidebar", "flexcol"] }` â€” a **dnd5e Application5e extension** that core ApplicationV2 silently ignores. Our parts (search, types, filters) render as direct children of `.window-content`. Our CSS grid targets `.compendium-browser .sidebar` which doesn't exist, so `grid-area: sidebar` has nothing to anchor to.

### R2: Tabs are icon-only with no text labels
Template renders `<a><i class="icon"></i></a>` â€” just icons with tooltips. In screenshot 1 these show as scattered icons. The dnd5e reference has text-labeled vertical tab buttons (e.g., "Features", "Races", "Spells").

### R3: Redundant H2 title inside content
Results template opens with `<h2>COMPENDIUM BROWSER</h2>`. Window chrome already shows this title.

### R4: Duplicate Configure Sources gear
Header template has the gear button AND `_renderFrame` injects one into window chrome. Only the chrome version should exist.

## Fixes (4 files, ordered by dependency)

### Fix 1: Merge sidebar parts into one part (solves R1)
- **Files:** `scripts/compendium-browser.js` (PARTS + context prep), `templates/browser-sidebar.hbs` (new)
- **Approach:** Replace 3 separate PARTS entries (search, types, filters) with ONE "sidebar" part containing a combined template. This creates the `.sidebar` wrapper div that the CSS grid needs.
- New PARTS entry:
  ```js
  sidebar: {
      id: "sidebar",
      classes: ["sidebar", "flexcol"],
      template: "modules/compendium-browser-bf/templates/browser-sidebar.hbs",
      templates: [
          "modules/compendium-browser-bf/templates/browser-sidebar-filter-set.hbs",
      ],
  }
  ```
- New sidebar template includes search + types + filters content inline.
- Merge `_prepareSearchContext`, `_prepareTypesContext`, `_prepareFiltersContext` into `_prepareSidebarContext`.
- Remove `case "search"`, `case "types"`, `case "filters"` from `_preparePartContext`, add `case "sidebar"`.
- Update `#activateListeners`/`#attachStaticListeners` to use new sidebar DOM selectors.

### Fix 2: Add text labels to tabs (solves R2)
- **File:** `templates/browser-tabs.hbs`
- **Approach:** Add `{{label}}` text next to each icon, style as vertical tab buttons with text.
- New template:
  ```hbs
  <nav data-group="primary">
      {{~#each tabs}}
      <a class="item control {{#if active}}active{{/if}}" data-tab="{{tab}}" ...>
          {{#if icon}}<i class="{{icon}}"></i>{{/if}}
          <span class="tab-label">{{localize label}}</span>
      </a>
      {{/each~}}
  </nav>
  ```
- Add CSS for `.tab-label` â€” small text below/next to icon.

### Fix 3: Remove redundant H2 (solves R3)
- **File:** `templates/browser-results.hbs`
- **Change:** Delete `<h2>COMPENDIUM BROWSER</h2>` line. Title comes from window chrome.

### Fix 4: Remove duplicate gear from header template (solves R4)
- **File:** `templates/browser-header.hbs`
- **Change:** Remove the `<button class="configure-sources">` block. Keep only the mode toggle. The gear in window chrome (from `_renderFrame`) is sufficient.

### Fix 5: Adjust header template for proper positioning
- **File:** `templates/browser-header.hbs`
- **Change:** The header currently has mode toggle + gear button in a flex row. After removing gear, just the mode toggle. Position it in the upper-left of the window content area (matching dnd5e's Advanced toggle location).
- Add CSS: position header absolutely at top of window-content, or place it in the grid's header area.

### Fix 6: Update CSS grid for 3-area layout
- **File:** `scss/module.scss`
- **Change:** The `.window-content` grid currently has `grid-template-areas: "sidebar results" "sidebar footer"`. After fixing R1, the sidebar actually exists. Also add a header area.
- New grid:
  ```css
  grid-template-areas: 
    "header header"
    "sidebar results"
    "sidebar footer";
  grid-template-rows: auto 1fr auto;
  ```
- Add `.window-content > .header { grid-area: header; }`

### Fix 7: Fix CSS selectors for sidebar content
- **File:** `scss/module.scss`
- **Change:** Current selectors like `.compendium-browser .sidebar [data-application-part="filters"]` won't work because the merged sidebar won't have `data-application-part="filters"`. Replace with class-based selectors (`.filter-section`, `.types-section`) or direct child selectors.
